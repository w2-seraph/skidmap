Usage

1. Establish key authentication to remote server used in "pushdorks.sh" (ssh-copy-id)
2. Copy contents of ./sqlmap to remote server home directory
3. Edit pushdorks.sh accordingly (remote server ip and user)
4. Generate dorks with provided tools
4. Run skidmap.sh

(optional) Examine dork crafting tools in ./keyword-generation
(optional) Examine .sh scripts used
(optional) Edit searx scripts / set or unset rotating proxy (highly prefered)
(optional) Check how tmux-shock.sh works and perhaps use it with cron
(final note) The sqlmap provided has been crafted to work against honeypots and other maladies that may cause it to hang
